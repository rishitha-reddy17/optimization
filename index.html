What is MSAL?

Microsoft Authentication Library — the secure engine powering this app's login. No passwords ever leave Microsoft's servers.

MSAL — Microsoft Authentication Library

v2.38

MSAL (em-sal) is Microsoft's official open-source JavaScript library. Its job: let your app sign users in via Azure AD / Microsoft accounts and receive a secure token — without your app ever seeing the password.

It acts as a trusted middleman. The user logs in on Microsoft's own servers. MSAL brings back a signed JWT proof that says "this person authenticated successfully." Your app never handles credentials directly.

OAuth 2.0 OpenID Connect Azure AD Multi-tenant JWT Tokens

How MSAL works in this app — step by step

1
User clicks "Sign in" — MSAL calls loginPopup(), opening Microsoft's secure login window.
2
Microsoft authenticates — credentials entered directly on Microsoft's servers. This app never receives or stores the password.
3
Token returned — Microsoft sends back an ID token (who you are) and an access token (what you're allowed to access).
4
MSAL caches the token in sessionStorage so the user stays logged in — no repeated logins.
5
Redirect to Azure Portal — the app reads your tenant from the token and opens portal.azure.com for your org.
